Graph comparing different sorting algorithm run times

The graph shows very poor performance of the Bubble Sort compared to Insertion and Selection Sort. When running the files with 10,000 words, it tooks upwards of 5000 ms to complete the task making it the highest run time compared to other sorts. It can also be noted that all three sorting algorithms failed when compiling 100,000 words and the program crashed. So, due to that, no average time is recorded for tests that have 100,000 words. Furthermore, this leads to the conclusion that these sorts are not great for large data sets. In addition, also insertion and selection sort work best when the data is nearly or completly ordered since the graph shows less average execution time for Ordered lists compared to Reverse-Ordered or Random lists.
